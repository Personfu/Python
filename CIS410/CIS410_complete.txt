Assignment 2: Docker Containerization
------------------------------------------------------------
Files: Dockerfile, docker-compose.yml

Dockerfile (Multi-stage):
  Stage 1 (builder): Node 20 Alpine, npm ci, npm run build
  Stage 2 (production): Copy dist + node_modules, non-root user,
    healthcheck every 30s, expose port 3000

Docker Compose (Full Stack):
  web: Custom build, port 3000, env vars for DB and Redis
  db: MySQL 8.0 with healthcheck, persistent volume, secrets
  cache: Redis 7 Alpine

Working Process:
  1. Multi-stage build reduces image from ~1.2GB to ~180MB
  2. Non-root user (appuser) for security best practice
  3. Healthcheck enables container orchestration auto-recovery
  4. docker-compose depends_on with condition ensures startup order

Assignment 3: CI/CD Pipeline
------------------------------------------------------------
File: deploy.yml

GitHub Actions workflow:
  Trigger: push/PR to main branch

  Job 1 — Test:
    - Checkout code
    - Setup Node.js 20 with npm cache
    - Install dependencies (npm ci)
    - Run linter
    - Run tests with coverage
    - Upload coverage report

  Job 2 — Build (needs: test, main branch only):
    - Build Docker image tagged with commit SHA
    - Run Docker Scout security scan

  Job 3 — Deploy (needs: build, main branch only):
    - Deploy to production environment

Assignment 4: Terraform Infrastructure as Code
------------------------------------------------------------
File: infrastructure.tf

RDS MySQL Instance:
  - db.t3.medium, 50GB encrypted storage
  - Multi-AZ for automatic failover
  - 30-day backup retention
  - Private subnet group, app-only security group

ECS Cluster & Service:
  - Fargate launch type (serverless containers)
  - 512 CPU / 1024 MB memory per task
  - 2 desired tasks with ALB load balancing
  - CloudWatch Logs integration
  - Container Insights enabled

Application Load Balancer:
  - Public-facing, HTTPS on port 443
  - TLS 1.3 security policy
  - Health check on /api/health endpoint
  - Target group with IP-based routing

Working Process:
  1. DB subnet group places RDS in private subnets only
  2. Security groups enforce least-privilege: ALB→App→DB chain
  3. ECS task definition includes environment variables and logging
  4. ALB listener terminates TLS and forwards to target group
