Four deliverables establish FLLC's frontend engineering
capability: semantic HTML, responsive CSS design systems,
JavaScript DOM frameworks, and REST API client patterns.

============================================================
Deliverable 1: Portfolio SPA
File: portfolio.html (420+ lines)
============================================================

Concepts:
  - Semantic HTML5: header, nav, main, section, footer
  - CSS custom properties (design tokens)
  - Flexbox layout for navigation and cards
  - CSS Grid for skills and projects sections
  - Sticky navigation with scroll behavior
  - Hero section with gradient background
  - Skills grid with animated progress bars
  - Project cards with hover effects and overlays
  - Experience timeline component
  - Contact section with form layout
  - Responsive breakpoints (mobile → desktop)

============================================================
Deliverable 2: DOM Manipulation Framework
File: dom_manipulation.js (320+ lines)
============================================================

Concepts:
  - DOM selection: getElementById, querySelector, querySelectorAll
  - Element creation factory function
  - Event handling: addEventListener for click, input, submit
  - Event delegation for dynamic element lists
  - classList API: add, remove, toggle, contains
  - localStorage helper functions (get, set, remove)
  - Complete TaskManager application:
    - Create, read, update, delete tasks
    - Filter by status (all, active, completed)
    - Task counter and statistics
    - Data persistence with localStorage
    - Modal dialog for confirmation

============================================================
Deliverable 3: Responsive Design System
File: responsive_design.css (330+ lines)
============================================================

Concepts:
  - Mobile-first methodology
  - CSS custom properties for design tokens
  - Fluid typography with clamp()
  - 4 breakpoints: 600px, 768px, 1024px, 1400px
  - CSS Grid dashboard layout (1 → 2 → 3 columns)
  - Flexbox for component-level layout
  - Dark/light theme toggle (prefers-color-scheme)
  - Print styles (@media print)
  - Accessibility: prefers-reduced-motion
  - Glassmorphism card effects (backdrop-filter)

============================================================
Deliverable 4: REST API Client
File: rest_api.js (270+ lines)
============================================================

Concepts:
  - fetch() API for HTTP requests
  - async/await pattern (replacing .then() chains)
  - All HTTP verbs: GET, POST, PUT, DELETE
  - Query parameter construction
  - AbortController for request timeout/cancellation
  - Retry logic with exponential backoff
  - Reusable APIClient class:
    - Base URL configuration
    - Default headers (auth, content-type)
    - Error handling with custom exceptions
    - Request/response interceptors
  - Dynamic DOM rendering from API data
  - Loading states and error UI
  - Pagination (next/previous page navigation)

Enterprise Application:
  Phase 09 integrates the API client with the production REST API.
  Design system CSS feeds the index.html coursework portal.
